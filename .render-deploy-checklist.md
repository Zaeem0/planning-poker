Use this checklist before deploying to ensure everything is ready.

## Pre-Deployment

- [ ] All code changes committed to git
- [ ] Code pushed to GitHub repository
- [ ] `.env.example` file is present and up-to-date
- [ ] No sensitive data in committed files
- [ ] `render.yaml` configuration is present
- [ ] `yarn.lock` file is committed to git
- [ ] Build command tested locally: `yarn install --frozen-lockfile && yarn build`
- [ ] Start command tested locally: `yarn start`

## Render Setup

- [ ] Render account created at [render.com](https://render.com)
- [ ] GitHub account connected to Render
- [ ] Repository access granted to Render

## Deployment

- [ ] New Web Service created in Render
- [ ] Correct repository selected
- [ ] `render.yaml` configuration detected and applied
- [ ] Environment variables reviewed (optional customization)
- [ ] First deployment initiated

## Post-Deployment Verification

- [ ] Build completed successfully (check Render logs)
- [ ] Service is running (status shows "Live")
- [ ] App accessible at Render URL
- [ ] Home page loads correctly
- [ ] Can create a new game
- [ ] Can join a game
- [ ] WebSocket connection works (check browser console)
- [ ] Voting functionality works
- [ ] Reveal votes works
- [ ] Reset functionality works
- [ ] Multiple users can join same game

## Optional Enhancements

- [ ] Custom domain configured (if needed)
- [ ] Upgraded to paid tier for always-on (if needed)
- [ ] Monitoring/alerts set up
- [ ] Custom `NEXT_PUBLIC_SOCKET_URL` configured (if needed)
- [ ] CORS_ORIGIN restricted to specific domain (for security)

## Troubleshooting

If something doesn't work:

1. Check Render logs for errors
2. Verify environment variables are set correctly
3. Test WebSocket connection in browser console
4. Review [DEPLOYMENT.md](./DEPLOYMENT.md) troubleshooting section
5. Check Render community forums
